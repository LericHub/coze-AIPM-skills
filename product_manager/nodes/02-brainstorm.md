# 02 - Brainstorm (头脑风暴)

## 职责

- 基于用户需求方向，主动抛出"虚"的功能骨架
- 发散思维与即时归纳循环
- 收集多种可能性并归纳为层级结构
- 不讨论技术实现细节，只聚焦功能方向
- 为 CLARIFY 做铺垫

---

## 执行逻辑

### 1. 开始头脑风暴

以开放、友好的态度引导用户，同时展示初步功能骨架：

```
欢迎进入头脑风暴阶段！

在这个阶段，我会基于您提供的需求方向，先抛出一个"虚"的功能骨架供您参考。
我们可以一起调整、补充，直到找到正确的方向。

功能骨架格式：
一级功能模块
  └─ 页面/二级功能
        └─ 子功能（接口级操作）

让我们开始吧！
```

### 2. 收集想法与即时归纳

#### 2.1 发散提问

使用开放式问题引导思维发散，不提供预设选项：

- "如果完全不考虑技术限制，您最想做的是什么？"
- "您期望这个功能达到什么效果？"
- "有没有什么疯狂的想法？"
- "您见过最让您惊艳的类似产品是什么？"

#### 2.2 即时归纳

当收集到足够信息后，立即归纳为层级结构展示给用户：

```
## 功能方向（草稿）

1. 价格预警
   - 价格预警仪表盘
     - 实时价格监控图表
     - 预警商品列表
     - 价格趋势图表
   - 价格预警规则设置
     - 阈值配置（高于/低于某价格）
     - 预警范围（商品/类目/店铺）
     - 通知方式（站内/短信/邮件）
   - 改价窗口
     - 单品调价
     - 批量调价
     - 调价历史

2. 库存预警
   ...

"这是您考虑的方向吗？需要调整或补充吗？"
```

#### 2.3 循环确认

根据用户反馈更新功能骨架：

- **用户调整**（增加模块、删除功能、修改层级）：重新归纳并展示
- **用户确认**（说"确认"、"没问题"）：进入完成流程
- **用户补充**：继续收集并归纳

### 3. 整理产出

当用户确认功能骨架后，整理最终产出：

```
✅ 功能策划完成！

## 功能方向（确认版）

1. 价格预警
   - 价格预警仪表盘（实时监控、预警列表、趋势图表）
   - 价格预警规则设置（阈值、范围、通知方式）
   - 改价窗口（单品/批量调价、历史记录）

2. 库存预警
   - 库存预警仪表盘（状态概览、预警列表、趋势图表）
   - 库存预警规则设置（阈值、维度、补货提醒）
   - 改库存窗口（调整、批量更新、记录）

3. 系统配置
   - 京东 API 授权
   - 数据刷新频率
   - 通知渠道配置

接下来进入需求澄清阶段，我们将深入探讨业务价值和边界。
```

## 目录初始化

在首次需要写文件时，创建完整的版本目录结构：

```
./output/V{version}/
├── brainstorm/
├── clarify/
├── analysis/
├── detail/
├── prototyping/
└── writing/
```

### 初始化时机

- CLARIFY 节点完成后，用户确认产出物时
- 智能体直接创建文件和目录，不指导用户手动创建
- 使用相对路径（以 `./` 开头），相对于用户工作目录

### 目录结构验证

创建后需验证：
- [ ] Memory.json 文件已创建
- [ ] output/V{version}/ 目录已创建
- [ ] 各节点子目录已创建

---

## 输出文件

- 文件名: `brainstorm.md`
- 路径: `./output/V{version}/brainstorm/brainstorm.md`
- 生成规则: 见 [01-router.md](01-router.md) 中的文件生成规则

### 拒绝技术讨论

- **WHEN** 用户询问技术实现（如数据库设计、接口规范、技术选型）
- **THEN** 智能体应告知"技术细节在后续阶段讨论，现在先聚焦功能方向"

### 保持功能抽象

- **WHEN** 智能体输出功能骨架时
- **THEN** 仅描述功能名称和用途，不包含技术实现信息

---

## 完成后

1. 保存产出到文件：`./output/V{version}/brainstorm/brainstorm.md`
2. 调用 manage.update_node_output("brainstorm", "output/V{version}/brainstorm/brainstorm.md")
3. 展示产出摘要（功能骨架确认版）
4. 等待用户确认
5. 确认后 → 调用 manage.update_node_status("brainstorm", "CONFIRM")
6. 确认后 → 调用 manage.transition_to_next_node()
7. 确认后 → 调用 manage.persist_memory()
8. 进入 Clarify 节点
