# 05 - Detail (详细设计)

## 职责
- 细化业务流程，绘制 Mermaid 流程图
- 事件列表细化
- 数据逻辑定义
- 埋点方案细化
- 页面维度的事件 + 异常输出
- 版本管理规范
- 文件保存规范

---

## 加载指引

### 前序节点产出物

从 Memory.json 获取 analysis 的 file 路径：
- 调用 manage.read_memory() 获取 node_list
- 从 node_list 中获取 analysis 的 file 字段
- 读取该文件内容

---

## 🚨 核心原则：文件保存与版本管理（最重要）

**⚠️ 强制要求：Step3 的所有产出物必须立即保存到文件，并严格进行版本管理！**

### 为什么重要？
1. **防丢失**：避免中断后内容丢失
2. **可追溯**：记录每个版本的完整内容
3. **版本对比**：方便用户对比不同版本
4. **流程衔接**：为 Step4 和 Step5 提供稳定的输入源

---

## 技能操作步骤

1. 读取 analysis 产出的文件内容
2. 按照输出模板组织
3. 使用 Mermaid 图表绘制业务流程图
4. 按页面维度细化事件列表和异常处理
5. **将生成的 Markdown 内容直接输出到用户消息中**（如果包含 Mermaid 代码块，需要特殊说明以便渲染）
6. 等待用户确认交付物

---

## 输出模板结构

### 1. 业务背景和需求概述

内容来源：node4 (analysis) 的完整内容

### 2. 高层级的业务流程图和页面流程图

#### 2.1 业务流程图

内容来源：node4 (analysis) 的业务流程输出格式（Mermaid 图表）

```
┌─────────────────────────────────────────────────────────────┐
│  格式要求：Mermaid 流程图或时序图                             │
│  示例：                                                   │

│    C -->|是| D[兑换商品]                                │
│    C -->|否| E[提示不足]                                │
```

#### 2.2 页面流程图

内容来源：node4 (analysis) 的页面流程输出格式（文字路径）

```
┌─────────────────────────────────────────────────────────────┐
│  格式要求：文字描述的页面跳转路径                            │
│  示例：                                                   │

└─────────────────────────────────────────────────────────────┘
```

### 3. 高层级各个端口的页面和功能列表

#### 3.1 C 端页面列表

内容来源：node4 (analysis) 的 C 端页面列表输出格式

#### 3.2 B 端页面列表

内容来源：node4 (analysis) 的 B 端页面列表输出格式

### 4. 埋点方案

内容来源：node4 (analysis) 的埋点方案，或根据业务场景新增输出格式

### 5. 以页面的维度，输出细分事件列表和细化需求

**结构：每个页面一个第 3 层标题**

#### 5.1 页面名称

**事件编号规范：页面编号-feat-序号**

##### 5.X.1 事件列表

```
| 事件编号 | 事件名称 | 绑定页面 | 触发节点 | 事件描述 |
|---------|---------|---------|---------|---------|
| /home/feat-01 | 查询会员信息 | 首页 | 载入时 | 用户登录后通过接口查询用户的积分和等级信息 |
| /home/feat-02 | 刷新商品列表 | 首页 | 点击刷新 | 用户点击刷新按钮，重新加载商品列表 |
| /home/feat-03 | 进入商品列表 | 首页 | 点击商品入口 | 用户点击商品入口，跳转到商品列表页 |
```

##### 5.X.2 事件逻辑

针对每个事件，详细说明：

1）业务流程图
   - 使用 Mermaid 展示该事件的完整业务流程

2）页面流程图（若涉及页面跳转）
   - 说明事件触发后的页面跳转路径

3）事件逻辑
   - 说明事件触发后链路，包括：
     - 触发什么接口
     - 获取/提交什么数据
     - 数据怎么展示/筛选/分组

4）异常场景和处理方案

**异常编号规范：事件编号-err序号**

```
| 异常场景 | 异常编号 | 判断方式 | 处理方案 |
|---------|---------|---------|---------|
| 未登录 | /home-feat-01-err01 | 检查 token 是否存在 | 显示登录按钮，引导用户登录 |
| 接口调用失败 | /home-feat-01-err02 | HTTP 状态码 != 200 | 显示"加载失败，请重试"提示 |
| 数据格式错误 | /home-feat-01-err03 | JSON 解析失败 | 显示默认的用户信息 |
```

## 输出文件

- 路径: `/AIPM/{project_name}_{date}_{order}/V{version}/detail/detail.md`
- 生成规则: 见 [01-router.md](01-router.md) 中的文件生成规则

---

## 完成后

1. 保存产出到文件：`/AIPM/{project_name}_{date}_{order}/V{version}/detail/detail.md`
2. 调用 manage.update_node_output("detail", "/AIPM/{project_name}_{date}_{order}/V{version}/detail/detail.md")
3. 展示产出摘要
4. 等待用户确认
5. 确认后 → 调用 manage.package_latest_version_output("{project_name}", "{date}", "{order}", "{version}", "detail")
6. 确认后 → 调用 manage.update_node_status("detail", "CONFIRM")
7. 确认后 → 调用 manage.transition_to_next_node()
8. 确认后 → 调用 manage.persist_memory()
9. 进入下一节点
