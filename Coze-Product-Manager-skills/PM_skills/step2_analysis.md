## 技能描述步骤一：需求分析

**🚨 关键规则：初步的 PRD 也必须非常详细，不能只是一个空框架。**

### 1.1 产出内容
根据步骤一的确认信息，直接生成一份详细的初步 **Markdown 格式**文档（存放在 prd/ 目录下，同时将文档内容以文本方式输出给用户）。这份初版 PRD 必须搭建好完整的文档结构骨架（参考 6.1 节的标准目录），并且必须深度填充以下部分：
1. **项目基本信息**
2. **需求背景与目标**：先说明需求背景，然后同列表的方式展示业务目标。业务目标必须包含四列表格（目标类型、描述、衡量指标、目标值）。
3. **识别用户类型和端口**：
    - 识别用户类型和端口。
    - 根据用户类型和端口，梳理用户在使用产品时的场景和核心流程。（先使用文字描述流程节点，如用户下单-》订单确认-》订单支付）
4. **识别业务对象**：用列表输出业务对象和关键属性（如用户、订单、商品等）。
5. **页面流程和页面列表**：
  - 用文字描述的方式输出B、C端页面流程，如：首页-》商品详情-》订单确认-》订单支付-》订单详情
  - 用列表输出B、C端页面列表，需要按照页面个能和用户操作两个维度描述。
  - 输出案例如下：
  --- markdown
  |页面编号|页面名称|功能描述|
  | ----------- | ----------- | ----------- |
  |/home|首页|展示用户积分和等级，展示商品列表，用户可以点击商品查看详情和发起兑换|
  ---
1. **埋点方案**：必须使用表格输出埋点场景的埋点的字段。
2. **异常场景**：对接系统返回的异常状态处理。（接口调用异常、接口返回业务异常）
*（后续章节如详细方案带原型的模块、整体流程图等可先留空，并注明待原型确认后补充）*
