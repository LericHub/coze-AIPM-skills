2. 解析根据文件内容和技能描述，按照输出模板输出交付物
3. CONTEXT_WRITE: [CURRENT_SNAPSHOT_WRITING] (将节点产出物写入上下文快照标签)
4. 完成PRD撰写

## 步骤一：产出最终版 PRD (内嵌原型与切片)

综合前三步的所有成果，输出最终的详细产品需求文档。

### 1.1 输出内容顺序
  1. **项目信息与版本记录** 
  2. **一、需求背景** (现状问题、为什么现在做)
  3. **二、需求目标** (目标类型、描述、衡量指标、目标值)
  4. **三、需求功能清单**(分端口、分模块输出页面列表和功能描述)
  5. **四、用户场景和业务流程** (输出step3_detail_design的`页面列表和页面功能`和`业务流程说明`部分)
  6. **五、数据追踪与埋点** (可选，若有则输出step3_detail_design的`埋点方案`部分)
  7. **六、异常与边界处理** (断网、空状态、无权限等)
  8. **七、页面设计**  (以端口-页面维度输出页面详细信息架构)
  9. **八、附件** (数据字典)
