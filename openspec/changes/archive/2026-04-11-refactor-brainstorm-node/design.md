## Context

当前 Brainstorm 节点（`product_manager/nodes/02-brainstorm.md`）的职责是自由发散思维、收集想法，但产出仅为简单的想法列表，缺乏结构化的功能策划引导。产品经理在需求方向确认后，期望获得一个初步的功能骨架来验证方向是否正确。

## Goals / Non-Goals

**Goals:**
- Brainstorm 节点在收集想法后，即时归纳为功能层级结构（一级模块 → 页面/二级 → 子功能）
- 产出的功能骨架是"虚"的，由产品经理决策是否实施
- 不讨论技术实现细节，只提供功能方向
- 为 Clarify 节点提供更结构化的输入

**Non-Goals:**
- 不修改 Clarify 节点逻辑
- 不改变节点流转顺序
- 不涉及技术实现方式讨论

## Decisions

1. **发散→归纳循环模式**
   - 采用"1-2轮发散 + 即时归纳"模式
   - 每轮发散后展示归纳结果，让用户确认方向

2. **层级结构设计**
   ```
   一级功能模块（如：用户管理）
     └─ 页面/二级功能（如：用户列表页）
           └─ 子功能/接口级（如：添加用户、编辑用户）
   ```

3. **话术模板**
   - 发散提问：开放式问题，不给选项
   - 归纳输出：结构化展示，"这些是你想的方向吗？"

## Risks / Trade-offs

- [Risk] 归纳过早可能限制用户思路 →  Mitigation: 明确告知"这是虚的方向，可以调整"
- [Risk] 层级粒度不统一 →  Mitigation: 提供示例模板，保持灵活性
