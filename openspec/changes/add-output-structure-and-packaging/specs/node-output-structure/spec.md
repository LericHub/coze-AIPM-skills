## ADDED Requirements

### Requirement: Prototyping output directory structure
The prototyping node SHALL output files in subdirectories organized by platform, with app/ and web/ folders for storing platform-specific prototypes.

#### Scenario: App prototype output
- **WHEN** node6 generates a prototype for a C端 (app) page
- **THEN** the prototype file SHALL be saved to `./output/V{version}/prototyping/app/`

#### Scenario: Web prototype output
- **WHEN** node6 generates a prototype for a B端 (web) page
- **THEN** the prototype file SHALL be saved to `./output/V{version}/prototyping/web/`

### Requirement: Writing node assets initialization
The writing node SHALL create an assets/ directory and copy logo.png from the project assets during output initialization.

#### Scenario: Assets directory creation
- **WHEN** node7 initializes its output directory
- **THEN** it SHALL create `./output/V{version}/writing/assets/`
- **AND** copy logo.png to `assets/logo.png`

### Requirement: PRD Viewer file path mapping
The PRD viewer generated by node7 SHALL reference prototype files using the new subdirectory structure.

#### Scenario: App page prototype path reference
- **WHEN** node7 generates page data for an app page
- **THEN** the prototypePath SHALL be `../prototyping/app/page_{name}.html`

#### Scenario: Web page prototype path reference
- **WHEN** node7 generates page data for a web page
- **THEN** the prototypePath SHALL be `../prototyping/web/page_{name}.html`
